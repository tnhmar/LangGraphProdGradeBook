\chapter{The Core Agent Loop}
\label{chap:core-loop}

\chapterquote{%
  A loop without an exit condition is not an agent; it is a fault.
  A loop without memory is not an agent; it is a script.
  The difference between a script and an agent lives entirely in
  what happens between one iteration and the next.%
}{Engineering AI Agents for Production, Volume~1}

\begin{learningbox}
After completing this chapter you will be able to:
\begin{itemize}
  \item Declare a \term{canonical state schema} whose fields serve all
        twenty-eight chapters without silent drift or undeclared keys.
  \item Build a \term{six-stage perception pipeline} as an independent
        compiled subgraph, with each stage unit-testable in isolation.
  \item Chain \term{four validation layers} between the reasoning node
        and tool dispatch so that hostile, incoherent, and irreversible
        requests are blocked before any side effect occurs.
  \item Replace the conventional single ``reflect'' node with a
        \term{seven-step deterministic review subgraph} and a
        \term{separate LLM-based reflection node}, preserving the
        boundary between world-state integration and strategic verdict.
  \item Wire all components into a compiled \code{StateGraph} with
        explicit terminal paths for \emph{completed}, \emph{escalated},
        and \emph{aborted} runs, each carrying distinct \code{RunState}
        values.
\end{itemize}
\end{learningbox}

\section{The Four-Phase Architecture}
\label{sec:four-phase}
% ─────────────────────────────────────────────────────────────────────────────

Volume~1 defines the \term{core agentic loop} as four phases that repeat
until a terminal condition is reached: \emph{Perceive}, \emph{Reason},
\emph{Act}, and \emph{Review}.  Each phase has a distinct contract.
Perceive normalises raw input into typed, trust-annotated observations.
Reason selects the next action from a typed union of decision types.
Act dispatches that action through a four-layer validation chain before
any side effect occurs.  Review integrates the result into the agent's
belief state, goal stack, and working memory without calling the model
again.

A fifth step, \emph{Reflect}, adds a strategic LLM verdict
(\textsc{continue}, \textsc{revise}, \textsc{retry}, \textsc{escalate},
or \textsc{terminate}) \emph{after} the deterministic review is
complete.  Volume~1 §2.2 is explicit that reflection and review are
separate responsibilities.  Conflating them into one node produces
agents that lose context when the LLM is unavailable and cannot be
tested deterministically.

\medskip
\noindent
This chapter implements all five phases in LangGraph.  Two of them
--- Perceive and Review --- are compiled as \emph{independent subgraphs}
that the outer graph adds as single named nodes.  This gives you
stage-level unit tests, independent versioning of each pipeline, and
clean observability boundaries.  The remaining three --- Reason, Act, and
Reflect --- are plain node functions.

\begin{notebox}
\textbf{Build order matters.}  The outer \code{StateGraph} is compiled
\emph{last}, after both subgraphs exist.  Running any \code{.invoke()}
or \code{.stream()} call on an uncompiled graph raises a
\code{RuntimeError} at the LangGraph level.  \code{compile()} is a
named architectural step, not a convenience method.
\end{notebox}

\section{The Canonical State Contract}
\label{sec:state-contract}
% ─────────────────────────────────────────────────────────────────────────────

Every node in every chapter reads from and writes to a single
\code{AgentState} \code{TypedDict}.  All fields used across all
twenty-eight chapters are declared \emph{here}, in \code{src/state.py},
annotated with the chapter that introduces them.  No chapter may access a
key that is not listed in this file.

\begin{warningbox}
\textbf{TypedDict is strict about undeclared keys.}  Writing
\code{state.get("new\_field")} without declaring \code{new\_field}
in the TypedDict is a silent type error under \texttt{mypy --strict}.
In production, it degrades to an undocumented contract that breaks
checkpoint compatibility every time a field is added.  The canonical
schema is the migration surface.  Treat it accordingly.
\end{warningbox}

\subsection{Resource tracking: \texttt{TokenBudget}}
\label{subsec:token-budget}

Volume~1 §2.3 treats the context window as a resource subject to explicit
ceilings.  A plain \code{int} field cannot simultaneously express tokens
consumed, tokens remaining, and cumulative cost in USD.  The
\component{TokenBudget} model carries all three:

\begin{lstlisting}[language=Python,
    caption={TokenBudget — structured budget tracking (src/state.py)},
    label={lst:token-budget}]
class TokenBudget(BaseModel):
    model_config = ConfigDict(frozen=True)
    ceiling:  int   = Field(ge=1)
    used:     int   = Field(ge=0, default=0)
    cost_usd: float = Field(ge=0.0, default=0.0)
    cost_cap: float | None = None      # None = no cost ceiling

    @property
    def remaining(self) -> int:
        return max(0, self.ceiling - self.used)

    def is_exhausted(self) -> bool:
        if self.used >= self.ceiling:
            return True
        if self.cost_cap is not None and self.cost_usd >= self.cost_cap:
            return True
        return False

    def add(self, tokens: int, cost: float = 0.0) -> "TokenBudget":
        """Return a new budget with tokens and cost accumulated."""
        return TokenBudget(
            ceiling  = self.ceiling,
            used     = self.used + tokens,
            cost_usd = round(self.cost_usd + cost, 6),
            cost_cap = self.cost_cap,
        )
\end{lstlisting}
\githubref{src.state.TokenBudget}

The model is \emph{frozen}: every update returns a new instance, making
the budget safe to store in LangGraph state reducers.  \code{is\_exhausted()}
covers both the token ceiling and the optional cost cap with a single call,
so every guard in Chapter~2 has one authoritative check.

\subsection{BDI fields: \texttt{Belief} and \texttt{Goal}}
\label{subsec:bdi-fields}

Volume~1 Chapter~9 grounds the agent's cognitive model in the
\term{Belief-Desire-Intention} (BDI) architecture.  Desires are expressed
as the root \code{goal} string; intentions as the mutable \code{goals}
hierarchy; beliefs as a list of \term{subject--predicate--object} triples.

\begin{lstlisting}[language=Python,
    caption={Belief and Goal — BDI state types (src/state.py)},
    label={lst:bdi-types}]
class Belief(BaseModel):
    model_config = ConfigDict(frozen=True)
    subject:    str
    predicate:  str
    object_:    str   = Field(alias="object")
    confidence: float = Field(ge=0.0, le=1.0, default=1.0)
    provenance: str   = ""   # e.g. "tool:search", "grounding_node"
    asserted_at: str  = Field(
        default_factory=lambda: datetime.now(timezone.utc).isoformat()
    )

class Goal(BaseModel):
    model_config = ConfigDict(frozen=True)
    id:             str       = Field(default_factory=lambda: str(uuid4()))
    description:    str
    parent_id:      str | None = None     # None = root goal
    status:         GoalStatus = GoalStatus.PENDING
    dependencies:   list[str]  = Field(default_factory=list)
    allocated_budget: int | None = None   # token sub-budget; None = inherit
    excluded_tools: list[str]  = Field(default_factory=list)
    required_tools: list[str]  = Field(default_factory=list)
\end{lstlisting}
\githubref{src.state.Belief}

The \code{provenance} field on \code{Belief} records which node asserted
the fact, making it straightforward to audit which stage of the perception
pipeline or which tool call produced a given belief.  The \code{Goal.parent\_id}
enables a tree of sub-goals: a planner node (Chapter~15) creates child
goals; the review subgraph (§\ref{sec:review-subgraph}) marks them
\code{COMPLETED} or \code{FAILED} one by one.

Listing~\ref{lst:agent-state} shows the twenty-four typed fields declared
in \code{AgentState}.  Fields are grouped by the chapter that first uses

\begin{tipbox}
\textbf{Only return changed keys from a node.}  LangGraph merges node
output dicts into the current state using the declared reducers.
Returning the full state dict from a node doubles memory use and defeats
the append-only invariant of audit lists.  Each node returns only the
keys it modifies.
\end{tipbox}

\section{The Perception Pipeline}
\label{sec:perceive-subgraph}
% ─────────────────────────────────────────────────────────────────────────────

Volume~1 Chapter~5 specifies perception as an \emph{explicit, composable
pipeline} with typed contracts at each stage boundary.  A single flat node
--- the conventional \code{perceive\_node(state)} function --- cannot be
tested stage by stage, cannot be versioned independently, and cannot surface
per-stage latency to the observability layer.

The \component{PerceiveSubgraph} compiles as a \code{StateGraph} in its
own right and is then added to the outer graph as a single node:

\begin{lstlisting}[language=Python,
    caption={Six-stage PerceiveSubgraph assembly (src/perceive.py)},
    label={lst:perceive-subgraph}]
def build_perceive_subgraph() -> StateGraph:
    sg = StateGraph(AgentState)

    sg.add_node("type_identifier",  type_identifier_node)
    sg.add_node("normalizer",        normalizer_node)
    sg.add_node("pii_redaction",     pii_redaction_node)
    sg.add_node("taint_annotator",   taint_annotator_node)
    sg.add_node("grounding",         grounding_node)
    sg.add_node("relevance_filter",  relevance_filter_node)

    sg.add_edge(START,               "type_identifier")
    sg.add_edge("type_identifier",   "normalizer")
    sg.add_edge("normalizer",        "pii_redaction")
    sg.add_edge("pii_redaction",     "taint_annotator")
    sg.add_edge("taint_annotator",   "grounding")
    sg.add_edge("grounding",         "relevance_filter")
    sg.add_edge("relevance_filter",   END)

    return sg   # caller calls sg.compile() inside build_agent_graph()

\subsection{Taint classification}
\label{subsec:taint}

The \component{TaintAnnotatorNode} calls \code{classify\_taint()} before
writing any input to working memory.  The function matches ten structural
prompt-injection patterns.  A \code{HOSTILE} label blocks all tool dispatch
downstream via the \component{TaintActionValidator} (§\ref{sec:action-node}).

\begin{lstlisting}[language=Python,
    caption={Taint classification — hostile pattern detection (src/perceive.py)},
    label={lst:classify-taint}]
_HOSTILE_PATTERNS = [
    re.compile(r"(?is)ignore\s+(all\s+)?(previous|prior|above)\s+"
               r"(instructions?|prompts?)"),
    re.compile(r"(?is)forget\s+(everything|all|your\s+instructions?)"),
    re.compile(r"(?is)(you\s+are\s+now|act\s+as|pretend\s+to\s+be)"
               r"\s+.{0,100}(assistant|ai|gpt|claude|llm)"),
    re.compile(r"(?is)\[\s*system\s*\]"),
    re.compile(r"(?is)<\s*system\s*>"),
    re.compile(r"(?is)###\s*(instruction|system|user)\s*:"),
    re.compile(r"(?is)\bDAN\b"),
    re.compile(r"(?is)\bOVERRIDE\s+INSTRUCTIONS?\b"),
]

def classify_taint(text: str | None) -> TaintLabel:
    if not text or not text.strip():
        return TaintLabel.CLEAN
    for pattern in _HOSTILE_PATTERNS:
        if pattern.search(text):
            return TaintLabel.HOSTILE
    return TaintLabel.EXTERNAL
\end{lstlisting}
\githubref{src.perceive.classify\_taint}

The \code{cycle\_count} increment also lives in \code{relevance\_filter\_node}
--- the last stage of the pipeline, which every input must pass through.
This is the \emph{only} place the counter advances.  Chapter~2's guards all
read it from state; none of them write it.

\begin{notebox}
\textbf{HOSTILE input terminates immediately.}
\code{relevance\_filter\_node} sets \code{run\_state = RunState.TERMINATED}
and returns without routing to Reason.  No LLM call is made.  The audit
trail in \code{cycle\_traces} records the early exit so operators can
inspect the triggering pattern in Langfuse.
\end{notebox}

\section{The Reasoning Node}
\label{sec:reason-node}
% ─────────────────────────────────────────────────────────────────────────────

The \component{ReasonNode} calls the model exactly once per cycle and
returns a typed \term{Decision} --- a sealed union of five concrete types.
Instructor enforces the schema and handles the retry loop for parse
failures.

\begin{lstlisting}[language=Python,
    caption={Decision union types (src/state.py)},
    label={lst:decision-types}]
class ToolRequest(BaseModel):
    model_config = ConfigDict(frozen=True, strict=True)
    tool_name:   str  = Field(min_length=1)
    tool_args:   dict[str, Any]
    rationale:   str  = Field(min_length=10)
    side_effect: SideEffectClass = SideEffectClass.READ_ONLY

class FinalAnswer(BaseModel):
    model_config = ConfigDict(frozen=True, strict=True)
    answer:     str
    confidence: float = Field(ge=0.0, le=1.0, default=1.0)
    sources:    list[str] = Field(default_factory=list)

class Escalate(BaseModel):
    model_config = ConfigDict(frozen=True)
    reason:  str
    context: dict[str, Any] = Field(default_factory=dict)

class AskClarification(BaseModel):
    model_config = ConfigDict(frozen=True)
    question: str

# ParallelToolRequests: list[ToolRequest] for concurrent dispatch (Ch20)
Decision = (ToolRequest | ParallelToolRequests
            | FinalAnswer | Escalate | AskClarification)
\end{lstlisting}
\githubref{src.state.Decision}

\medskip
Before \code{reason\_node} returns a \code{Decision}, it runs
\component{GoalCoherencePlanValidator} --- a plan-level check that the
proposed action is coherent with the current \code{GoalStack}.  This is
\emph{not} the same as the output-schema validation Instructor handles
internally.  It runs after Instructor has produced a structurally valid
\code{Decision} and asks a semantic question: \emph{does this action make
sense given what the agent is trying to achieve?}

If the validator returns \code{needs\_correction}, \code{revision\_count}
increments.  Exceeding \code{policy.max\_retries} produces a
\code{PLAN\_INCOHERENT} termination before any tool is invoked.

\begin{tipbox}
\textbf{The \code{SideEffectClass} field on \code{ToolRequest} matters
immediately.}  It flows into the four-layer validator chain in the action
node and determines whether the call requires human approval (Chapter~21)
or can auto-proceed.  Declare it at reasoning time, not at dispatch time.
\end{tipbox}

\section{The Action Node and Four-Layer Validation}
\label{sec:action-node}
% ─────────────────────────────────────────────────────────────────────────────

Between the \component{ReasonNode} and the actual tool invocation sit four
independent \code{ActionValidator} implementations, composed in a
\component{CompositeActionValidator}.  Each layer asks a different question
and produces a \code{ValidationVerdict} on failure.  The chain short-circuits
at the first failure.

\begin{table}[htbp]
\centering
\caption{Four-layer ActionValidator chain}
\label{tab:validators}
\renewcommand{\arraystretch}{1.4}
\begin{tabularx}{\linewidth}{@{} l l P{5.5cm} @{}}
\toprule
\textbf{Layer} & \textbf{Validator} & \textbf{Question} \\
\midrule
1 & \code{SchemaActionValidator}
  & Do the tool arguments match \code{ToolContract.input\_schema}? \\
2 & \code{SemanticActionValidator}
  & Is this call coherent with the current \code{GoalStatus}
    and not in the Goal's \code{excluded\_tools} list? \\
3 & \code{SafetyActionValidator}
  & Does \code{SideEffectClass} require approval before execution? \\
4 & \code{TaintActionValidator}
  & Does any \code{HOSTILE}-labelled entry in working memory
    indicate an active injection attack? \\
\bottomrule
\end{tabularx}
\end{table}

\begin{lstlisting}[language=Python,
    caption={CompositeActionValidator — chains all four layers (src/validators.py)},
    label={lst:composite-validator}]
class CompositeActionValidator:
    def __init__(
        self,
        validators: list[ActionValidator] | None = None,
    ) -> None:
        self._validators = validators or [
            SchemaActionValidator(),
            SemanticActionValidator(),
            SafetyActionValidator(),
            TaintActionValidator(),
        ]

    def validate(
        self,
        call:     ToolRequest,
        contract: ToolContract,
        state:    AgentState,
        attempt:  int = 0,
    ) -> ValidationVerdict:
        for v in self._validators:
            verdict = v.validate(call, contract, state)
            if not verdict.passed:
                return ValidationVerdict(
                    passed        = False,
                    error_message = verdict.error_message,
                    error_field   = verdict.error_field,
                    layer         = verdict.layer,
                    attempt       = attempt,
                )
        return ValidationVerdict.ok()
\end{lstlisting}
\githubref{src.validators.CompositeActionValidator}

Layer~3 (\component{SafetyActionValidator}) maps \code{SideEffectClass}
values to \code{RiskClassification}.  When a tool is classified
\code{IRREVERSIBLE} or \code{HIGH\_BLAST\_RADIUS}, the validator sets
\code{error\_field = "requires\_approval"}.  The action node reads this
sentinel, constructs an \component{ApprovalPacket} (Chapter~21), and
transitions to \code{RunState.SUSPENDED\_HITL} instead of dispatching.
The tool call waits for human input.  Volume~1 §17.3 calls this the
structured approval surface.

\begin{warningbox}
\textbf{Layer~4 blocks the entire session, not just the current call.}
A single \code{HOSTILE}-labelled working-memory entry freezes all tool
dispatch until the working memory is cleared.  The rationale is that if
an injection has reached working memory, any subsequent tool call may
carry tainted arguments, making partial blocking insufficient.  Clearing
working memory requires an explicit operator action.
\end{warningbox}

% ─────────────────────────────────────────────────────────────────────────────
\section{The Review Subgraph}
\label{sec:review-subgraph}
% ─────────────────────────────────────────────────────────────────────────────

After a tool returns, the agent must integrate the result into its world
model before reasoning again.  Volume~1 §2.2 specifies this as a sequence
of deterministic updates --- no model call is involved.  The
\component{ReviewSubgraph} implements this in seven steps, each compiled
as an independent node:

\begin{lstlisting}[language=Python,
    caption={ReviewSubgraph assembly — seven deterministic steps (src/review.py)},
    label={lst:review-subgraph}]
def build_review_subgraph() -> StateGraph:
    sg = StateGraph(AgentState)

    sg.add_node("taint_classification",   taint_classification_node)
    sg.add_node("wm_update",              wm_update_node)
    sg.add_node("goal_stack_update",       goal_stack_update_node)
    sg.add_node("belief_update",           belief_state_update_node)
    sg.add_node("lt_writeback",            lt_writeback_node)
    sg.add_node("post_action_validation",  post_action_validation_node)
    sg.add_node("termination_check",       termination_check_node)

    sg.add_edge(START,                     "taint_classification")
    sg.add_edge("taint_classification",    "wm_update")
    sg.add_edge("wm_update",               "goal_stack_update")
    sg.add_edge("goal_stack_update",        "belief_update")
    sg.add_edge("belief_update",            "lt_writeback")
    sg.add_edge("lt_writeback",             "post_action_validation")
    sg.add_edge("post_action_validation",   "termination_check")
    sg.add_edge("termination_check",         END)

    return sg
\end{lstlisting}
\githubref{src.review.build\_review\_subgraph}

\medskip

Each step has a narrow, testable responsibility:

\begin{itemize}
  \item \textbf{TaintClassification} --- re-runs \code{classify\_taint()}
        on the action result.  A tool that calls an external API may return
        content that contains an injection attempt.
  \item \textbf{WMUpdate} --- writes the result to working memory with the
        correct \code{WorkingMemoryTier}.  Successful, information-rich
        results are tagged \code{ACTIVE}; error output is tagged
        \code{COMPRESSED}.
  \item \textbf{GoalStackUpdate} --- marks the current sub-goal
        \code{COMPLETED} when the decision was a \code{FinalAnswer}, or
        \code{FAILED} when the action errored.
  \item \textbf{BeliefUpdate} --- asserts new S-P-O \code{Belief} triples
        derived from the result, tagged with the tool name as provenance.
  \item \textbf{LTWriteback} --- calls \code{mem0.add()} only when the
        action's \code{SideEffectClass} is \code{NON\_IDEMPOTENT} or
        higher.  Read-only results are not persisted to long-term memory.
        (Chapter~8 expands this step into the full five-check write gate.)
  \item \textbf{PostActionValidation} --- increments
        \code{consecutive\_failures} on error and transitions to
        \code{PLAN\_INCOHERENT} termination when the revision budget is
        exhausted.
  \item \textbf{TerminationCheck} --- checks all resource ceilings
        (\code{max\_cycles}, \code{token\_ceiling}, \code{cost\_cap}) and
        the stagnation detector before returning \code{MEMORY\_UPDATE}
        to the outer graph.  The outer graph interprets \code{MEMORY\_UPDATE}
        as ``proceed to Reflect''.
\end{itemize}

\begin{notebox}
\textbf{No model call occurs in the review subgraph.}  Every step is a
pure function of state.  This means the review subgraph can be tested
with synthetic state dictionaries and no API credentials, and it will
always produce the same output for the same input --- a property the
reflection node cannot provide.
\end{notebox}

% ─────────────────────────────────────────────────────────────────────────────
\section{The Reflection Node}
\label{sec:reflect-node}
% ─────────────────────────────────────────────────────────────────────────────

The \component{ReflectNode} is the single place in the loop where the
model evaluates \emph{strategic} progress.  It runs only when
\component{TerminationCheckNode} has confirmed the run is still live.
Its output is a \code{ReflectionOutput} carrying a five-value verdict
and a \term{goal delta} signal.

\begin{lstlisting}[language=Python,
    caption={ReflectionOutput and routing verdicts (src/state.py)},
    label={lst:reflection-output}]
class ReflectVerdict(str, Enum):
    CONTINUE  = "CONTINUE"   # cycle again from Perceive
    REVISE    = "REVISE"     # plan changed; cycle from Perceive with new intent
    RETRY     = "RETRY"      # execution failed; re-enter Reason without Perceive
    ESCALATE  = "ESCALATE"   # cannot proceed; surface to human or abort
    TERMINATE = "TERMINATE"  # goal achieved; write FinalAnswer

class ReflectionOutput(BaseModel):
    model_config = ConfigDict(frozen=True)
    verdict:    ReflectVerdict
    goal_delta: Literal["ADVANCED", "NO_CHANGE", "REGRESSED"] = "NO_CHANGE"
    reasoning:  str  = ""
    confidence: float = Field(ge=0.0, le=1.0, default=1.0)
\end{lstlisting}
\githubref{src.state.ReflectionOutput}

The \code{goal\_delta} field drives Chapter~2's loop detectors.  When
\code{goal\_delta} is \code{NO\_CHANGE} for \code{GuardConfig.stagnation\_window}
consecutive cycles, the stagnation detector fires before the next
\code{ReflectNode} call.

\medskip
\noindent
\textbf{Routing} after \code{ReflectNode} uses \code{goal\_hash\_history}
computed from the SHA-256 of the current goal-stack shape, not from
embedding similarity.  The structural-hash approach requires no extra
model call and is reproducible across restarts.

\begin{warningbox}
\textbf{\textsc{escalate} and \textsc{terminate} must route to different
nodes.}  \code{TERMINATE} means the goal was achieved --- the run exits
with \code{RunState.COMPLETED} and a \code{FinalAnswer}.
\code{ESCALATE} means the agent cannot proceed --- the run exits with
\code{RunState.TERMINATED}, a \code{dead\_letter} snapshot, and
\code{TerminationReason.ESCALATED}.  Collapsing both to \code{END}
destroys the distinction downstream systems rely on to route completed
runs to delivery queues and escalated runs to human review queues.
\end{warningbox}

\section{Graph Wiring and \texttt{compile()}}
\label{sec:graph-wiring}
% ─────────────────────────────────────────────────────────────────────────────

\code{build\_agent\_graph()} assembles the outer graph by adding the two
compiled subgraphs as single nodes, then wiring the remaining node
functions and their conditional edges.  \code{compile()} is called once,
at the bottom of the factory function.

\begin{lstlisting}[language=Python,
    caption={Outer graph wiring and compile() (src/graph.py)},
    label={lst:graph-wiring}]
def build_agent_graph(
    checkpointer: BaseCheckpointSaver | None = None,
    policy:       TerminationPolicy   | None = None,
    guard:        GuardConfig         | None = None,
) -> CompiledStateGraph:
    _policy = policy or TerminationPolicy()
    _guard  = guard  or GuardConfig()

    perceive_sg = build_perceive_subgraph()
    review_sg   = build_review_subgraph()

    g = StateGraph(AgentState)

    # Subgraphs added as single named nodes
    g.add_node("perceive",   perceive_sg)   # 6-stage pipeline
    g.add_node("reason",     reason_node)
    g.add_node("action",     action_node)
    g.add_node("review",     review_sg)     # 7-step review
    g.add_node("reflect",    reflect_node)
    g.add_node("escalation", escalation_node)

    # Edges
    g.add_edge(START,      "perceive")
    g.add_edge("perceive", "reason")
    g.add_edge("reason",   "action")
    g.add_edge("action",   "review")

    g.add_conditional_edges("review",  route_after_review, {
        "reflect":  "reflect",
        "terminal": END,
    })
    g.add_conditional_edges("reflect", route_reflect_verdict, {
        "perceive":   "perceive",
        "reason":     "reason",
        "escalation": "escalation",
        "terminal":   END,
    })
    g.add_edge("escalation", END)

    # recursion_limit = max_cycles × 4 gives room for recovery branches
    return g.compile(
        checkpointer=checkpointer,
        recursion_limit=_policy.max_cycles * 4,
    )
\end{lstlisting}
\githubref{src.graph.build\_agent\_graph}

The routing function \code{route\_reflect\_verdict} is the
\emph{application-level} cycle ceiling check --- Layer~1 of the
two-layer guard that Chapter~2 describes in full.  LangGraph's own
\code{recursion\_limit} is Layer~2.  Both must be present; neither alone
is sufficient.

\begin{lstlisting}[language=Python,
    caption={route\_reflect\_verdict — two-layer cycle guard (src/graph.py)},
    label={lst:route-reflect}]
def route_reflect_verdict(
    state:  AgentState,
    config: RunnableConfig,
) -> str:
    # Layer 1: application-level ceiling check (fires before reading verdict)
    policy_dict = state.get("_policy") or {}
    max_cycles  = policy_dict.get("max_cycles", 20)
    if (state.get("cycle_count") or 0) >= max_cycles:
        return "terminal"

    budget = state.get("token_budget")
    if budget and budget.is_exhausted():
        return "terminal"

    run_state = state.get("run_state", RunState.PLANNING)
    if run_state in {RunState.COMPLETED, RunState.TERMINATED, RunState.ABORTED}:
        return "terminal"

    reflect = state.get("reflect_output")
    if not reflect:
        return "perceive"     # default: continue the loop

    return {
        ReflectVerdict.CONTINUE:  "perceive",
        ReflectVerdict.REVISE:    "perceive",
        ReflectVerdict.RETRY:     "reason",
        ReflectVerdict.ESCALATE:  "escalation",
        ReflectVerdict.TERMINATE: "terminal",
    }.get(reflect.verdict, "perceive")
\end{lstlisting}
\githubref{src.graph.route\_reflect\_verdict}

\section{The Fourteen-State RunState Machine}
\label{sec:run-state}
% ─────────────────────────────────────────────────────────────────────────────

\code{RunState} carries fourteen values covering every phase the agent
can be in, from initialisation to each distinct terminal outcome.
Volume~1 Chapter~8 is explicit that \code{DEGRADED} must not collapse into
\code{TERMINATED}: the former is a recoverable provider fault; the latter
is a permanent exit.

\begin{lstlisting}[language=Python,
    caption={RunState — fourteen lifecycle values (src/state.py)},
    label={lst:run-state}]
class RunState(str, Enum):
    # --- live states ---
    INITIALIZED     = "INITIALIZED"    # before first cycle
    VALIDATING      = "VALIDATING"     # resource pre-check
    PLANNING        = "PLANNING"       # reason + decision
    MODEL_CALL      = "MODEL_CALL"     # LLM in-flight
    TOOL_EXECUTION  = "TOOL_EXECUTION" # action dispatch
    MEMORY_UPDATE   = "MEMORY_UPDATE"  # post-action review
    RESPONDING      = "RESPONDING"     # assembling answer
    AWAITING_RESULT = "AWAITING_RESULT"# async tool pending
    DEGRADED        = "DEGRADED"       # provider down; fallback active
    SUSPENDED_HITL  = "SUSPENDED_HITL" # interrupt() fired
    WAITING_FOR_JOB = "WAITING_FOR_JOB"# Celery / async runner
    # --- terminal states ---
    COMPLETED  = "COMPLETED"   # goal achieved normally
    ABORTED    = "ABORTED"     # degraded beyond recovery
    TERMINATED = "TERMINATED"  # escalated / budget / cycles

    def is_terminal(self) -> bool:
        return self in {RunState.COMPLETED, RunState.ABORTED,
                        RunState.TERMINATED}
\end{lstlisting}
\githubref{src.state.RunState}

The transition \code{DEGRADED} $\to$ \code{ABORTED} is handled in
Chapter~5 when all provider fallback tiers have been exhausted.
\code{SUSPENDED\_HITL} is handled in Chapter~21 when an
\code{interrupt()} fires on an \code{IRREVERSIBLE} tool call.
\code{WAITING\_FOR\_JOB} is the state a Celery-backed run holds between
submission and pickup (Chapter~22).

\begin{takeawaybox}[Chapter~1 Key Takeaways]
\begin{itemize}
  \item \textbf{Declare the full state schema first.}  Every field used
        across all twenty-eight chapters lives in \code{AgentState} in
        \code{src/state.py}, annotated with its introducing chapter.
        Undeclared keys are silent type errors.

  \item \textbf{Perception and Review are subgraphs, not nodes.}
        Compiling each as an independent \code{StateGraph} gives you
        stage-level unit tests, independent versioning, and clean
        Langfuse child spans.

  \item \textbf{Four validation layers separate reasoning from execution.}
        Schema, semantic, safety, and taint checks run between the
        \code{Decision} and the tool invocation.  The chain short-circuits
        at the first failure.  Layer~4 blocks the \emph{session} on a
        hostile taint, not just the current call.

  \item \textbf{Review is deterministic; Reflection uses the model.}
        The seven-step review subgraph requires no LLM call and is fully
        testable with synthetic state.  The reflection node adds the
        strategic verdict separately, after the world-state update is
        complete.

  \item \textbf{\textsc{escalate} and \textsc{terminate} are different
        terminal paths.}  \code{COMPLETED} means success; \code{TERMINATED}
        means escalated or resource-exhausted; \code{ABORTED} means the
        provider degraded beyond recovery.  All three must route to distinct
        downstream systems.

  \item \textbf{Observability is structural, not additive.}  \code{@observe}
        on every node and \code{LangfuseCallbackHandler} in
        \code{RunnableConfig} provide full trace coverage without
        modifying node logic.
\end{itemize}
\end{takeawaybox}

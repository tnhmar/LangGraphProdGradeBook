\chapterquote{%
  A provider outage is not an edge case.  It is a Tuesday.
  The agent that survives is the one that planned for Tuesday before
  it shipped, not the one patched after the first incident.%
}{Engineering AI Agents for Production, Volume~1}

\begin{learningbox}
After completing this chapter you will be able to:
\begin{itemize}
  \item Externalise every model parameter into a single
        \code{Pydantic Settings} object so no model identifier is ever
        hardcoded at a call site.
  \item Implement a three-state \term{circuit breaker} per model that
        stops sending requests to a failing provider rather than waiting
        for every request to time out individually.
  \item Build a typed \term{fallback cascade} that tries each configured
        provider in order and surfaces which one ultimately served the
        request.
  \item Track token usage and cost from LiteLLM's callback hook and
        enforce a hard budget ceiling that halts the agent before an
        invoice surprises anyone.
  \item Apply the \code{DEGRADED} $\to$ \code{ABORTED} transition
        correctly when every configured provider has failed.
\end{itemize}
\end{learningbox}

\section{Externalised Configuration}
\label{sec:llm-settings}
% ─────────────────────────────────────────────────────────────────────────────

Every value that determines which model the agent calls lives in one
\code{Pydantic Settings} object loaded from environment variables.

\begin{lstlisting}[language=Python,
    caption={LLMSettings --- single source of model configuration
             (src/llm.py)},
    label={lst:llm-settings}]
class LLMSettings(BaseSettings):
    model_config = SettingsConfigDict(
        env_prefix="LLM_", env_nested_delimiter="__"
    )
    primary_model:   str = "gpt-4o-2024-11-20"
    fallback_models: list[str] = Field(default=[
        "claude-3-5-sonnet-20241022",
        "gpt-4o-mini-2024-07-18",
    ])
    instructor_mode: str   = "TOOLS"   # TOOLS | JSON | MD_JSON
    temperature:     float = Field(default=0.0, ge=0.0, le=2.0)
    max_tokens:      int   = Field(default=4096, ge=1)
\end{lstlisting}
\githubref{src.llm.LLMSettings}

\begin{warningbox}
\textbf{Every model identifier is version-pinned.}  \code{"gpt-4o-2024-11-20"},
not \code{"gpt-4o"}.  Provider-side silent model updates --- a vendor
repointing an unpinned alias overnight --- are a real production incident
category.  A pinned identifier makes a model upgrade a deliberate code
change with its own test cycle.
\end{warningbox}

% ─────────────────────────────────────────────────────────────────────────────
\section{The Circuit Breaker}
\label{sec:circuit-breaker}
% ─────────────────────────────────────────────────────────────────────────────

A circuit breaker stops the agent from discovering that a provider is
down one slow timeout at a time.  Once a provider has failed
\code{failure\_threshold} consecutive times, the breaker opens and every
subsequent call fails immediately --- no network round-trip --- until a
cooldown has elapsed.

\begin{lstlisting}[language=Python,
    caption={CircuitBreaker --- three-state machine (src/llm.py)},
    label={lst:circuit-breaker}]
class CircuitBreaker:
    def __init__(self, failure_threshold=5, cooldown_s=60.0, name="default"):
        self._state     = _CBState.CLOSED
        self._failures  = 0
        self._threshold = failure_threshold
        self._cooldown  = cooldown_s
        self._opened_at: float | None = None
        self._lock      = threading.Lock()

    @property
    def state(self) -> _CBState:
        with self._lock:
            if self._state == _CBState.OPEN:
                if time.monotonic() - (self._opened_at or 0) >= self._cooldown:
                    self._state = _CBState.HALF_OPEN
            return self._state

    def call(self, fn: Callable, *args, **kwargs) -> Any:
        if self.state == _CBState.OPEN:
            raise CircuitOpenError(f"Circuit '{self._name}' is OPEN")
        try:
            result = fn(*args, **kwargs)
            self._on_success()
            return result
        except Exception:
            self._on_failure()
            raise
\end{lstlisting}
\githubref{src.llm.CircuitBreaker}

Transition sequence: \code{CLOSED} $\to$ \code{OPEN} after
\code{failure\_threshold} failures; \code{OPEN} $\to$ \code{HALF\_OPEN}
once \code{cooldown\_s} elapses (checked lazily inside the \code{state}
property, not on a background timer); one probe decides
\code{HALF\_OPEN} $\to$ \code{CLOSED} (success) or
\code{HALF\_OPEN} $\to$ \code{OPEN} (failure).

\begin{tipbox}
\textbf{One breaker per model, not one for the whole router.}
\code{LLMRouter} keeps \code{self.\_cbs: dict[str, CircuitBreaker]}.
A primary-model outage opens only the primary's breaker; fallback
models remain individually available.
\end{tipbox}

\section{The Fallback Cascade}
\label{sec:fallback-cascade}
% ─────────────────────────────────────────────────────────────────────────────

\code{LLMRouter.complete()} walks the configured model list in order,
stopping at the first model that returns successfully.

\begin{lstlisting}[language=Python,
    caption={LLMRouter.complete --- fallback iteration (src/llm.py)},
    label={lst:fallback-cascade}]
def complete(self, messages: list[dict], max_retries: int = 3, **kwargs) -> Any:
    all_models = [self._cfg.primary_model, *self._cfg.fallback_models]

    for model in all_models:
        cb = self._cbs.get(model, CircuitBreaker(name=model))
        try:
            result = self._call_with_retry(
                model, messages, max_retries, cb, **kwargs
            )
            langfuse_context.update_current_observation(metadata={
                "model_used": model,
                "fallback":   model != self._cfg.primary_model,
            })
            return result
        except (CircuitOpenError, litellm.exceptions.APIConnectionError):
            continue                       # try next model
        except BudgetExceededError:
            raise                          # never silently swallow
        except Exception:
            continue

    raise RuntimeError("All models exhausted; no LLM call succeeded.")
\end{lstlisting}
\githubref{src.llm.LLMRouter.complete}

Each attempt is also wrapped in a Tenacity retry (\code{\_call\_with\_retry})
for transient errors.  Two retry layers for two failure timescales: Tenacity
handles a request likely to succeed on the second try within the same model;
the fallback loop handles a provider that will not succeed at all this window.

\begin{warningbox}
\textbf{\code{BudgetExceededError} is re-raised, never caught by the
fallback loop.}  Trying a different provider does not help when the cost
cap is exhausted --- the next provider costs money too.
\end{warningbox}

\section{Cost Tracking and the Budget Circuit Breaker}
\label{sec:cost-tracking}
% ─────────────────────────────────────────────────────────────────────────────

LiteLLM exposes a \code{success\_callback} that fires after every
completed call.  Registering here is the only reliable way to capture
token counts across every provider without writing a provider-specific
parser.

\begin{lstlisting}[language=Python,
    caption={Budget callback and drain (src/llm.py)},
    label={lst:budget-callback}]
_BUDGET_QUEUE: list[dict] = []   # process-global; see note below

def _budget_callback(kwargs, completion_response,
                     start_time, end_time) -> None:
    usage = getattr(completion_response, "usage", None)
    if not usage:
        return
    cost = litellm.completion_cost(completion_response=completion_response)
    _BUDGET_QUEUE.append({
        "tokens": getattr(usage, "total_tokens", 0),
        "cost":   cost or 0.0,
    })

def drain_budget_into_state(state: dict) -> dict:
    budget = state.get("token_budget")
    if not budget:
        return {}
    total_tokens = total_cost = 0
    while _BUDGET_QUEUE:
        item = _BUDGET_QUEUE.pop(0)
        total_tokens += item["tokens"]
        total_cost   += item["cost"]
    if total_tokens == 0:
        return {}
    new_budget = budget.add(total_tokens, total_cost)
    if new_budget.is_exhausted():
        raise BudgetExceededError(
            f"Budget exceeded: {new_budget.used}/{new_budget.ceiling} tokens"
        )
    return {"token_budget": new_budget}
\end{lstlisting}
\githubref{src.llm.drain\_budget\_into\_state}

\begin{notebox}
\textbf{The queue is process-global; the budget is per-run.}
\code{\_BUDGET\_QUEUE} accumulates across all concurrent runs in the same
process.  This is safe because \code{drain\_budget\_into\_state()} is
called synchronously, immediately after the specific LLM call it is
measuring, before any other run's call can interleave on a
single-threaded event loop.  A multi-threaded deployment would need a
per-run queue.
\end{notebox}

\section{DEGRADED: The State Between Working and Giving Up}
\label{sec:degraded}
% ─────────────────────────────────────────────────────────────────────────────

When the fallback cascade exhausts every configured model, Volume~1
Chapter~8 requires this be treated as \code{RunState.DEGRADED}, not
immediate termination --- a future cycle might recover if a provider
comes back.

\begin{lstlisting}[language=Python,
    caption={handle\_all\_providers\_failed --- DEGRADED then ABORTED
             (src/llm.py)},
    label={lst:degraded}]
def handle_all_providers_failed(state: dict) -> dict:
    if state.get("run_state") == RunState.DEGRADED:
        # Already degraded once — now give up
        return {
            "run_state":          RunState.ABORTED,
            "termination_reason": TerminationReason.FAILURE_ESCALATION,
        }
    return {"run_state": RunState.DEGRADED}
\end{lstlisting}
\githubref{src.llm.handle\_all\_providers\_failed}

The first total failure in a cycle transitions to \code{DEGRADED}; the
outer graph routes back to \code{perceive} for one more attempt.  Only
when \emph{the very next} cycle also exhausts every provider does the
run transition \code{DEGRADED} $\to$ \code{ABORTED}.

\begin{warningbox}
\textbf{\code{ABORTED} is reachable only through \code{DEGRADED}.}
No code path sets \code{RunState.ABORTED} directly from a live state.
\code{ABORTED} means the agent tried, partially recovered, tried again,
and still failed --- a meaningfully different operational signal from a
single transient blip, and downstream alerting should treat it as more
severe.
\end{warningbox}

\section{Applying the Subgraph Principle to \texttt{LLMRouter.complete()}}
\label{sec:ch5-subgraph}
% ─────────────────────────────────────────────────────────────────────────────

\code{complete()} iterates over up to three models, each wrapped in a
circuit breaker and a Tenacity retry --- real network I/O across multiple
attempts.  The strongest false-positive so far, purely on the strength of
condition~4 (I/O boundary).  Applying all four tests:

\begin{takeawaybox}[Subgraph Principle applied: verdict — stays a plain method]
\begin{enumerate}
  \item \emph{Three or more steps?}  Yes --- up to three model attempts.
  \item \emph{Distinct failure modes worth separate spans?}
        \textbf{No.}  Every attempt in the loop has the \emph{same}
        failure mode: this model was unavailable.  They differ only in
        which model name is attached.  PerceiveSubgraph's stages catch
        categorically different problems; this loop retries the same
        category of problem repeatedly.
  \item \emph{Communicate through \code{AgentState}?}  \textbf{No.}
        The loop variable \code{model} and the per-attempt circuit
        breaker are local to the \code{for} loop.  Nothing about which
        model is currently being tried is visible in \code{AgentState}.
  \item \emph{I/O boundary, or resumability value between attempts?}
        I/O: yes.  Resumability: \textbf{no.}  There is no operational
        value in resuming ``after trying the primary but before trying
        fallback~1.''  A restart simply re-enters \code{complete()} from
        the top.
\end{enumerate}
Conditions~2 and~3 both fail decisively.  This is the clearest
illustration that \textbf{I/O presence alone is not sufficient} for
subgraph treatment.
\end{takeawaybox}

def test_circuit_opens_after_threshold_failures(breaker):
    for _ in range(3):
        with pytest.raises(ValueError):
            breaker.call(lambda: (_ for _ in ()).throw(ValueError("boom")))
    assert breaker.state == _CBState.OPEN

def test_circuit_closed_after_successful_probe(breaker):
    # open it
    for _ in range(3):
        with pytest.raises(ValueError):
            breaker.call(lambda: (_ for _ in ()).throw(ValueError("boom")))
    time.sleep(0.02)   # cooldown_s=0.01 in fixture
    breaker.call(lambda: "ok")
    assert breaker.state == _CBState.CLOSED

    rs, reason = _handle_all_providers_failed(RunState.DEGRADED)

\begin{takeawaybox}[Chapter~5 Key Takeaways]
\begin{itemize}
  \item \textbf{One config object, zero hardcoded model names.}
        \code{LLMSettings} is the single place that names a model;
        version-pinned identifiers make provider upgrades a deliberate,
        testable change.

  \item \textbf{One circuit breaker per model.}  A primary-model outage
        opens only the primary's breaker; fallback models remain
        individually available.

  \item \textbf{Fallback and Tenacity retry solve different problems.}
        Tenacity handles a single model's transient errors; the fallback
        loop handles a model that is entirely down for the window.

  \item \textbf{\code{BudgetExceededError} is never swallowed by the
        fallback loop.}  Switching providers cannot fix an exhausted cost
        cap.

  \item \textbf{\code{DEGRADED} is not a synonym for failure.}
        \code{ABORTED} is reachable only via \code{DEGRADED}, and only
        after a second consecutive total failure.

  \item \textbf{I/O presence alone does not qualify a function for
        subgraph treatment.}  \code{LLMRouter.complete()} performs real
        network I/O across multiple attempts and still correctly stays a
        plain method, because its steps share one failure mode and
        communicate through local variables.
\end{itemize}
\end{takeawaybox}
